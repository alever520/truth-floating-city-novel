# 小说创作检查清单

## 📋 使用说明

**每次创作章节时,必须完整执行以下检查清单。**

完成一项,在`[ ]`中填入`✅`,并记录完成时间。

**版本**: 基于 novel-writer skill v2.2.0

---

## ✅ 步骤0: 接收创作任务

- [ ] 用户明确要求创作第X章
- [ ] 记录任务接收时间: ____________

---

## ✅ 步骤1: 读取全局大纲与上章预告

### 1.1 读取全局大纲
- [ ] 读取 `references/outline.md` (全局大纲)
- [ ] 确认本章在全书中的位置
- [ ] 确认与前后章节的衔接关系
- [ ] 记录完成时间: ____________

### 1.2 读取上章预告（如适用）
- [ ] 如果是第001章 → 跳过此步骤
- [ ] 如果是第002章及以后 → 读取上一章的"📖 下章预告"
- [ ] 整合预告与大纲，确认一致性
- [ ] 记录完成时间: ____________

### 1.3 ⚠️ 上下文管理检查点（关键）
- [ ] **保持原文在上下文中** → 步骤3大纲确认前不清空
- [ ] 理由：LLM需要原文才能准确生成大纲

---

## ✅ 步骤2: 读取人物卡

### 2.1 读取人物卡
- [ ] 读取 `references/characters.md`
- [ ] 确认本章出场的已有人物设定
- [ ] 确认本章新增人物是否需要补充设定
- [ ] 记录完成时间: ____________

### 2.2 ⚠️ 上下文管理检查点（关键）
- [ ] **保持原文在上下文中** → 步骤3大纲确认前不清空
- [ ] 理由：生成大纲时需要参考完整人物设定

---

## ✅ 步骤3: 创作章节大纲

### 3.1 生成章节大纲
- [ ] 基于步骤1、2读取的内容生成大纲
- [ ] 大纲包含七个模块：
  - [ ] 核心任务
  - [ ] 世界观锚点
  - [ ] 场景规划（含关键对话示例）
  - [ ] 人物动向
  - [ ] 节奏安排
  - [ ] 伏笔处理
  - [ ] 预计字数
- [ ] 记录完成时间: ____________

### 3.2 大纲确认循环
- [ ] 向用户展示大纲
- [ ] 等待用户确认
- [ ] 如有修改意见 → 修改 → 再次确认
- [ ] 记录确认时间: ____________

### 3.3 大纲写入
- [ ] 将确认的大纲写入 `chapter-outlines/第XXX章-标题-大纲.md`
- [ ] 检查是否需要更新 `references/outline.md`
- [ ] 如有更新 → 向用户展示改动位置/内容/原因
- [ ] 记录完成时间: ____________

### 3.4 ⚠️ 上下文管理检查点（关键）
**大纲确认并写入后，执行以下清理：**
- [ ] **清空步骤1的上下文**：全局大纲、上章预告（约5000字）
- [ ] **清空步骤2的上下文**：人物卡（约3000字）
- [ ] **保留当前章节大纲**：已提炼的核心信息（约1000字）
- [ ] **Token节省**：8000字 → 1000字（节省约87%）

---

## ✅ 步骤4: 读取写作指导与读者反馈（⚠️ 必须执行）

### 4.1 读取并提炼写作规范
- [ ] 读取 `references/writing-guide.md` 全文
- [ ] **按大类提炼**：每个大类（如"章节结构"/"节奏控制"/"内容创作"等）各提炼1-3条
- [ ] **相关性判断**：根据章节大纲判断哪些规则与本章最相关
- [ ] **总量控制**：所有大类合计不超过15条关键规则
- [ ] 记录完成时间: ____________
- [ ] 记录提炼规则数量: _____条

### 4.2 读取并筛选读者反馈
- [ ] 读取 `references/reader-feedback.md` 全文
- [ ] **状态筛选**：只关注"待处理"状态的条目
- [ ] **相关性判断**：筛选与本章主题/场景/人物相关的反馈
- [ ] 记录完成时间: ____________
- [ ] 记录待处理反馈数量: _____条

### 4.3 向用户汇报
- [ ] 按大类展示提炼的关键规则
- [ ] 展示筛选后的待处理反馈
- [ ] 等待用户确认批准开始创作
- [ ] 记录汇报时间: ____________

### 4.4 ⚠️ 上下文管理检查点（关键）
**汇报后，执行以下清理：**
- [ ] **清空原文**：writing-guide.md（约5000字）+ reader-feedback.md（约2000字）
- [ ] **保留提炼内容**：15条关键规则（约500字）
- [ ] **Token节省**：7000字 → 500字（节省约93%）

---

## ✅ 步骤5: 创作章节正文

### 5.1 上下文检查（⚠️ 关键）
**确认当前上下文状态：**
- [ ] **必须保持**：
  - [ ] 当前章节大纲（约1000字）
  - [ ] 步骤4提炼的15条关键规则（约500字）
  - [ ] 总计：约1500字
- [ ] **已清空内容**：
  - [ ] 全局大纲、上章预告（步骤1）
  - [ ] 人物卡原文（步骤2）
  - [ ] 完整写作规范、完整反馈记录（步骤4）
  - [ ] 总计：约15000字
- [ ] **Token节省效果**：15000字 → 1500字（节省90%）

### 5.2 创作准备
- [ ] 回顾章节大纲（已在上下文中，无需重新读取）
- [ ] 确认字数目标: ________字
- [ ] 记录开始创作时间: ____________

### 5.3 创作执行
- [ ] 严格按章节大纲推进剧情
- [ ] 场景顺序、节奏安排、伏笔处理均参照大纲执行
- [ ] 遵守步骤4提炼的15条关键规则
- [ ] 遵守步骤4筛选的待处理反馈
- [ ] **按需读取**：
  - [ ] 写到角色互动时 → 临时读取 `references/characters.md` 对应角色卡
  - [ ] 需要确认世界观设定时 → 临时读取相关参考文档
  - [ ] 用完即可遗忘，无需持续占用
- [ ] 记录创作完成时间: ____________
- [ ] 实际字数: ________字

---

## ✅ 步骤6: 自检清单（⚠️ 必须全部通过）

### 6.1 大纲执行(4项)
- [ ] 场景序列与大纲一致（或有说明的合理偏差）
- [ ] 节奏安排符合大纲（高潮位置、收尾节奏）
- [ ] 伏笔处理符合大纲（新埋/回收/延续）
- [ ] 字数在大纲预计范围内

### 6.2 剧情(3项)
- [ ] 符合全局大纲规划
- [ ] 推进主线/支线
- [ ] 人物行为符合性格

### 6.3 语言(8项)
- [ ] 避免短句堆砌
- [ ] 流畅自然
- [ ] 避免重复用词
- [ ] 对话符合人设
- [ ] 删除元叙事表述（"狗血的""套路""网文"等）
- [ ] 内心独白符合人物身份（避免不符合人设的粗口、网络流行语）
- [ ] 感官描写有具体修饰词（"灼烧的""嗡嗡的"等）
- [ ] 比喻精确且符合世界观

### 6.4 节奏(7项)
- [ ] 有张有弛
- [ ] 场景转换自然（⚠️ 同章节内禁用`---`分割线）
- [ ] 爽点分布合理
- [ ] 信息密度适中
- [ ] 删除读者能自己理解的推理过程
- [ ] 合并重复的对话和问题
- [ ] 用疑问替代生硬的断言

### 6.5 细节(4项)
- [ ] 道具有世界观功能（如恒温法阵）
- [ ] 人物描写包含脸型、眼睛颜色等具体细节
- [ ] 数字形式统一（对话中用阿拉伯数字）
- [ ] 生理反应符合情境逻辑（刚醒来不能"好多了"）

### 6.6 读者反馈(3项)
- [ ] 已处理待解决问题
- [ ] 未重复已批评的错误
- [ ] 采纳合理建议

### 6.7 技术(3项)
- [ ] 文件命名规范: 第XXX章-标题.md（使用三位数编号）
- [ ] Markdown 格式正确
- [ ] 字数符合要求

### 6.8 自检通过率
- [ ] 计算通过率: _____/32项 = _____%
- [ ] 要求: 必须≥95% (最多允许1-2项未达标)
- [ ] 记录自检完成时间: ____________

## 🔴 关键提醒

### 必须执行的步骤
1. **步骤3.4上下文管理检查点**: 大纲确认后清空步骤1、2的上下文
2. **步骤4.1智能提炼**: 从writing-guide.md每个大类提炼1-3条规则（总计≤15条）
3. **步骤4.2筛选反馈**: 只关注"待处理"且与本章相关的反馈
4. **步骤4.4上下文管理检查点**: 汇报后清空原文，保留提炼规则
5. **步骤5.1上下文检查**: 确认保持内容约1500字（节省90% token）

### 禁止事项
1. **禁止使用`---`分割线**: 同一章节内场景转换必须用自然过渡
2. **禁止跳过上下文管理检查点**: 每个检查点都是关键节点
3. **禁止完整保持原文**: 步骤4必须提炼，不能直接保持5000字规范

### 关键数字
- 字数目标: 每章约8000字
- 自检通过率: ≥95% (32/32项或31/32项)
- 提炼规则数: ≤15条
- 上下文占用: ≤1500字（创作时）
- Token节省: 约90%

## 🔄 上下文管理流程图

```
步骤1、2 → 读取大纲、人物卡
【保持】8000字原文
    ↓
步骤3 → 生成章节大纲
【确认后】执行检查点3.4
    ↓
【清空】步骤1、2原文（8000字）
【保留】章节大纲（1000字）
    ↓
步骤4 → 读取并智能提炼规则
【读取】7000字原文
【提炼】15条规则（500字）
    ↓
【汇报后】执行检查点4.4
【清空】原文（7000字）
【保留】提炼规则（500字）
    ↓
步骤5 → 创作正文
【保持】大纲（1000字）+ 规则（500字）
【按需】临时读取人物卡片段
    ↓
完成 → Token节省90%
```
