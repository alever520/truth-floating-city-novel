# 小说创作检查清单

## 📋 使用说明

**每次创作章节时,必须完整执行以下检查清单。**

完成一项,在`[ ]`中填入`✅`,并记录完成时间。

---

## ✅ 步骤0: 接收创作任务

- [ ] 用户明确要求创作第X章
- [ ] 记录任务接收时间: ____________

---

## ✅ 步骤1: 读取全局大纲

### 1.1 读取文件
- [ ] 读取 `references/outline.md` (全局大纲)
- [ ] 记录完成时间: ____________

### 1.2 确认位置
- [ ] 确认本章在全书中的位置
- [ ] 确认与前后章节的衔接关系

---

## ✅ 步骤2: 读取/生成章节大纲

### 2.1 检查大纲是否存在
- [ ] 检查 `chapter-outlines/第XXX章-标题-大纲.md` 是否存在
- [ ] 如果存在 → 读取
- [ ] 如果不存在 → 生成新大纲并向用户确认

### 2.2 大纲确认
- [ ] 向用户展示大纲
- [ ] 等待用户确认
- [ ] 如有修改意见 → 修改 → 再次确认
- [ ] 记录确认时间: ____________

---

## ✅ 步骤3: 大纲写入

- [ ] 将确认的大纲写入 `chapter-outlines/第XXX章-标题-大纲.md`
- [ ] 检查是否需要更新 `references/outline.md`
- [ ] 记录完成时间: ____________

---

## ✅ 步骤4: 读取写作指导与读者反馈（⚠️ 必须执行）

### 4.1 读取写作指导
- [ ] 读取 `references/writing-guide.md`
- [ ] 记录完成时间: ____________
- [ ] 记录关键规范: ____________________

### 4.2 读取读者反馈
- [ ] 读取 `references/reader-feedback.md`
- [ ] 重点关注"待处理"状态的条目
- [ ] 记录完成时间: ____________
- [ ] 记录待处理问题: ____________________

---

## ✅ 步骤5: 创作章节正文

### 5.1 创作准备
- [ ] 回顾章节大纲 `chapter-outlines/第XXX章-标题-大纲.md`
- [ ] 确认字数目标: ________字

### 5.2 创作执行
- [ ] 严格按大纲推进剧情
- [ ] 场景顺序与大纲一致
- [ ] 遵守步骤3.5确认的章节要求
- [ ] 遵守步骤4读取的写作指导与读者反馈
- [ ] 记录创作完成时间: ____________
- [ ] 实际字数: ________字

---

## ✅ 步骤6: 自检清单（⚠️ 必须全部通过）

### 6.1 大纲执行(4项)
- [ ] 场景序列与大纲一致
- [ ] 节奏安排符合大纲
- [ ] 伏笔处理符合大纲
- [ ] 字数在大纲预计范围内

### 6.2 剧情(3项)
- [ ] 符合全局大纲规划
- [ ] 推进主线/支线
- [ ] 人物行为符合性格

### 6.3 语言(8项)
- [ ] 避免短句堆砌
- [ ] 流畅自然
- [ ] 避免重复用词
- [ ] 对话符合人设
- [ ] 删除元叙事表述
- [ ] 内心独白符合人物身份
- [ ] 感官描写有具体修饰词
- [ ] 比喻精确且符合世界观

### 6.4 节奏(4项)
- [ ] 有张有弛
- [ ] 场景转换自然（⚠️ 无`---`分割线）
- [ ] 爽点分布合理
- [ ] 信息密度适中

### 6.5 细节(4项)
- [ ] 道具有世界观功能
- [ ] 人物描写包含脸型、眼睛颜色等具体细节
- [ ] 数字形式统一
- [ ] 生理反应符合情境逻辑

### 6.6 读者反馈(2项)
- [ ] 已处理待解决问题
- [ ] 未重复已批评的错误

### 6.7 技术(3项)
- [ ] 文件命名规范: 第XXX章-标题.md
- [ ] Markdown 格式正确
- [ ] 字数符合要求

### 6.8 自检通过率
- [ ] 计算通过率: _____/28项 = _____%
- [ ] 要求: 必须100%通过
- [ ] 记录自检完成时间: ____________

---

## ✅ 步骤7: 提交到GitHub

### 7.1 Git操作
- [ ] `git add chapters/第XXX章-标题.md`
- [ ] `git commit -m "[提交信息]"`
- [ ] `git push origin main`
- [ ] 记录commit hash: ____________

### 7.2 提交信息规范
新创作:
```
新增第X章: [标题] (XXXX字)

核心剧情: [简述]
伏笔处理: [新埋/回收]
自检通过率: XX/28项
```

重新创作:
```
重新创作第X章: [改进要点]

- 应用最新规范
- 优化[具体内容]

字数: XXXX字（旧版YYYY字）
自检通过率: XX/28项
```

## 🔴 关键提醒

1. **步骤4必须执行**: 读取 `references/writing-guide.md` 和 `references/reader-feedback.md`
2. **禁止使用`---`分割线**: 同一章节内场景转换必须用自然过渡
3. **字数目标**: 每章约8000字
4. **自检通过率**: 必须100%(28/28项)
5. **步骤8必须执行**: 补充完整的章节末尾信息
